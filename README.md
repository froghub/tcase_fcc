## Как запустить
1. Копируем .env.example в .env  . Заполняем
2. Если есть docker - `make up` (`docker compose up -d` если нет make)
3. Остановить `make down` (`docker compose down` если нет make)

## Как проверить
Для начала мигрируем и заливаем тестовые данные:  
```bash
make yii migrate/fresh && make yii seed/all
```

 - Либо использовать rest_api_test.http - это файл rest клиента phpstorm
 - Либо запустить тест (`make test`), но он совсем простой - проверяет создаются ли сущности и отрабатывает ли главная логика - напоминания
 - Использовать сторонние клиенты. Логин тестового пользователя m@m.m, пароль 123

Команда для проверки оповещений - `make yii notification/check`  
Писать будет процесс - в stdout, уведомления - runtime/logs/notification.log  

Для проверки расписания ставится задача в cron: */10 * * * * /путь/к/make/make yii notification/check  
Но так как контейнер облегченный - в нем нет кронтаба, потому заработает только на "живом" чем то  


Кратко об архитектуре:  

repositories/ReminderRepository - инкапсулирует сложную логику доступа к БД  
services/ReminderService - занимается оркестрацией получения расписания, проверки и рассылки  
services/NotifierInterface и LogNotifier - контракт и реализация уведомлятеля. В данном случае канал - лог  
tests/Functional/NotificationTest - тест проверки консольной команды. Api не проверяет, хотя и ловит ошибки  валидации моделей  
Задачи покрыть все тестами не было, потому только его сделал для удобства

Ну а миграции,модели,контроллеры - все там где обычно кладут в yii
